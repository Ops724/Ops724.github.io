VMware Fusion 系列，DHCP 静态绑定

硬件：Apple M4
系统：Mac OS Tahoe 26.3
VMware Fusion：Professional 26H1（25388279）

当我们创建了多台虚机的本地环境时，集群内一个稳定的网络状态是必不可少的，通过 VMware Fusion DHCP 配置绑定 MAC 地址是比较推荐的方式，统一在一个配置文件里管理全局，省去了在多台虚机间切换配置静态地址的繁琐。

VMware Fusion DHCP 添加静态绑定策略

  1. 备份原始配置文件
sudo cp /Library/Preferences/VMware\ Fusion/vmnet8/dhcpd.conf \
         /Library/Preferences/VMware\ Fusion/vmnet8/dhcpd.conf.backup
  1. sudo vim /Library/Preferences/VMware\ Fusion/vmnet8/dhcpd.conf。
# 增加静态绑定规则
host vm {
    hardware ethernet 00:0c:29:4d:c5:ed;  # 虚拟机的MAC地址
    fixed-address 172.16.29.128;          # 固定的IP地址
    option domain-name "";                # 域名解析服务器（可选）
    option routers 172.16.29.2;           # 网关地址（可选）
}

两个 option 配置项可选，不指定则使用 DHCP 服务全局配置。

  1. 重启 VMware Fusion 网络
sudo /Applications/VMware\ Fusion.app/Contents/Library/vmnet-cli --stop
sudo /Applications/VMware\ Fusion.app/Contents/Library/vmnet-cli --start

规则生效。
